79 /**
80  * @brief Initialise (but not start) the Audio Input Subsystem.
81  * @details Returns FALSE for an invalid channel or other invalid parameter.
82  *
83  * @param[in] channel The channel to convert. Can be set from 0 to GAUDIN_NUM_CHANNELS - 1.
84  * @param[in] frequency The sample frequency
85  * @param[in] buffer The static buffer to put the samples into.
86  * @param[in] bufcount The total number of conversions that will fit in the buffer.
87  * @param[in] samplesPerEvent The number of conversions to do before returning an event.
88  *
89  * @note Only one channel is active at a time. If an audio input is running it will be stopped.
90  * The Event subsystem is disconnected from the audio subsystem and any binary semaphore
91  * event is forgotten.
92  * @note Some channels may be stereo channels which return twice as much sample data with
93  * the left and right channel data interleaved. Other channels may be mono channels.
94  * Where stereo channels exist it would be common for the low level driver to also
95  * offer the left and right channels separately.
96  * @note Due to a bug in Chibi-OS countPerEvent must be even if using the GADC low level audio driver.
97  * If bufcount is not evenly divisable by countPerEvent, the remainder must also be even.
98  * This requirement may not apply to other GAUDIN drivers.
99  * @note The number of samples for stereo devices will be double the number of conversions.
100  * Make sure you allocate your buffers large enough. Each channel is then interleaved
101  * into the provided buffer. Note 'bufcount' and 'countPerEvent' parameters describe the
102  * number of conversions not the number of samples.
103  * @note The buffer is circular. When the end of the buffer is reached it will start
104  * putting data into the beginning of the buffer again.
105  * @note The event listener must process the event (and the data in it) before the
106  * next event occurs. If not, the following event will be lost.
107  * @note If bufcount is evenly divisable by countPerEvent, then every event will return
108  * countPerEvent conversions. If bufcount is not evenly divisable, it will return
109  * a block of samples containing less than countPerEvent samples when it reaches the
110  * end of the buffer.
111  *
112  * @return FALSE if invalid channel or parameter
113  *
114  * @api
115  */
116 bool_t gaudinInit(uint16_t channel, uint32_t frequency, audin_sample_t *buffer, size_t bufcount, size_t samplesPerEvent);
