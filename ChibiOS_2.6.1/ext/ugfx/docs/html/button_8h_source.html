62  * @param[in] gb The GButtonObject structure to initialise. If this is NULL the structure is dynamically allocated.
63  * @param[in] pInit The initialisation parameters
64  *
65  * @note The drawing color and the background color get set to the current defaults. If you haven't called
66  * @p gwinSetDefaultColor() or @p gwinSetDefaultBgColor() then these are White and Black respectively.

89 /**
90  * @brief Some custom button drawing routines
91  * @details These function may be passed to @p gwinSetCustomDraw() to get different button drawing styles
92  *
93  * @param[in] gw The widget object (in this case a button)
94  * @param[in] param A parameter passed in from the user
95  *
96  * @note In your custom button drawing function you may optionally call these
97  * standard functions and then draw your extra details on top.
98  * @note The standard functions below ignore the param parameter except for @p gwinButtonDraw_Image().
99  * @note The image custom draw function @p gwinButtonDraw_Image() uses param to pass in the gdispImage pointer.
100  * The image must be already opened before calling @p gwinSetCustomDraw(). The image should be 3
101  * times the height of the button. The button image is repeated 3 times vertically, the first (top) for
102  * the "up" image, the 2nd for the "down" image, and the third (bottom) image for the disabled state. If
103  * the disabled state is never going to be used then the image can be just 2 times the button height.
104  * No checking is done to compare the size of the button to the size of the image.
105  * Note text is drawn on top of the image.
106  * @note These custom drawing routines don't have to worry about setting clipping as the framework
107  * sets clipping to the object window prior to calling these routines.
108  *
109  * @api
110  * @{
111  */
