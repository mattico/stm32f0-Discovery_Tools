8 /**
9  * @file include/gtimer/gtimer.h
10  *
11  * @addtogroup GTIMER
12  *
13  * @brief Module which provides software based timers for user-space applications
14  *
15  * @details The reason why ChibiOS/GFX has it's own timer abstraction is because
16  * virtual timers provided by ChibiOS/RT are interrupt context only.
17  * While great for what they are designed for, they make coding of the input
18  * drivers much more complex.
19  * For non-performance critical drivers like these input drivers, it would also
20  * hog an in-ordinate amount of critical (interrupt locked) system time.
21  * This contrary to the goals of a real-time operating system. So a user-land
22  * (thread based) timer mechanism is also required.
23  *
24  * @pre GFX_USE_GTIMER must be set to TRUE in your gfxconf.h
25  *
26  * @{
27  */

79 /**
80  * @brief Set a timer going or alter its properties if it is already going.
81  *
82  * @param[in] pt Pointer to a GTimer structure
83  * @param[in] fn The callback function
84  * @param[in] param The parameter to pass to the callback function
85  * @param[in] periodic Is the timer a periodic timer? FALSE is a once-only timer.
86  * @param[in] millisec The timer period. The following special values are allowed:
87  * TIME_IMMEDIATE causes the callback function to be called asap.
88  * A periodic timer with this value will fire once only.
89  * TIME_INFINITE never timeout (unless triggered by gtimerJab or gtimerJabI)
90  *
91  * @note If the timer is already active its properties are updated with the new parameters.
92  * The current period will be immediately canceled (without the callback function being
93  * called) and the timer will be restart with the new timer properties.
94  * @note The callback function should be careful not to over-run the thread stack.
95  * Define a new value for the macro GTIME_THREAD_STACK_SIZE if you want to
96  * change the default size.
97  * @note The callback function should return as quickly as possible as all
98  * timer callbacks are performed by a single thread. If a callback function
99  * takes too long it could affect the timer response for other timers.
100  * @note A timer callback function is not a replacement for a dedicated thread if the
101  * function wants to perform computationally expensive stuff.
102  * @note As the callback function is called on GTIMER's thread, the function must make sure it uses
103  * appropriate synchronisation controls such as semaphores or mutexes around any data
104  * structures it shares with other threads such as the main application thread.
105  *
106  * @api
107  */
108 void gtimerStart(GTimer *pt, GTimerFunction fn, void *param, bool_t periodic, delaytime_t millisec);
