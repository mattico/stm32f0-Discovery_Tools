size_t gadc_lld_samples_per_conversion ( uint32_t  physdev)

Get the number of samples in a conversion.

Calculates and returns the number of samples per conversion for the specified physdev.

Note
A physdev describing a mono device would return 1, a stereo device would return 2. For most ADC's physdev is a bitmap so it is only a matter of counting the bits.
Parameters
[in]physdevA value passed to describe which physical ADC devices/channels to use.
Returns
Number of samples of the convesion

void gadc_lld_start_timer ( uint32_t  physdev,
uint32_t  frequency 
)

Start a periodic timer for high frequency conversions.

Parameters
[in]physdevA value passed to describe which physical ADC devices/channels to use.
[in]frequencyThe frequency to create ADC conversions
Note
The exact meaning of physdev is hardware dependent. It describes the channels the will be used later on when a "timer" conversion is actually scheduled.
It is assumed that the timer is capable of free-running even when the ADC is stopped or doing something else.

When a timer interrupt occurs a conversion should start if these is a "timer" conversion active.

Note
If the ADC is stopped, doesn't have a "timer" conversion active or is currently executing a non-timer conversion then the interrupt can be ignored other than (optionally) incrementing the GADC_Timer_Missed variable.
Function Class:
Normal API, this function can be invoked by regular system threads but not from within a lock zone.

void gadc_lld_adc_timerI ( GadcLldTimerData *  pgtd)

Start a "timer" conversion.

Starts a series of conversions triggered by the timer.

Parameters
[in]pgtdContains the parameters for the timer conversion.
Note
The exact meaning of physdev is hardware dependent. It is likely described in the drivers gadc_lld_config.h
Some versions of ChibiOS actually call the callback function more than once, once at the half-way point and once on completion. The high level code handles this.
The driver should call GADC_ISR_CompleteI() when it completes the operation (or at the half-way point), or GAD_ISR_ErrorI() on an error.
The high level code ensures that this is not called while a non-timer conversion is in progress
Function Class:
This is an I-Class API, this function can be invoked from within a system lock zone by both threads and interrupt handlers.
